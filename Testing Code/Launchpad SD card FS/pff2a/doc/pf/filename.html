Format of the path names

The path name format on the Petit FatFs module is similer to MS-DOS as follows.

 "[/]directory/file"

The Petit FatFs module supports only 8.3 format file name. The sub directories are separated with a /. The path name is terminated with a NUL character, control character or white space. Extended characters (0x80-0xFF) are not allowed for the path name on ASCII only configuration (_CODE_PAGE == 1).

The Petit FatFs module does not have a concept of current directory like OS oriented file system. All objects on the volume are always specified in full path name following from the root directory. Dot names are not available. Heading separator is ignored and it can be exist or omitted.
